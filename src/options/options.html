How to write URL rewrite rules.

How Rules Work

Each rule has: Name (label), Pattern (regex matched against the full URL), Modification (text to insert/replace), and Mode:

  • Before — modification is inserted before each regex match
  • Replace — each match is replaced by the modification; supports $1, $2 capture group references
  • After — modification is appended after each match

Rules run top-to-bottom; the output of each rule feeds the next. Invalid regex patterns fall back to literal string matching.

Examples

NamePatternModificationModeEffect
Strip UTM params[?&]utm_[^&]*(empty)replaceRemoves ?utm_source=… tracking params
Force HTTPS^http://https://replaceUpgrades HTTP → HTTPS
Add CDN prefix(static\.example\.com)cdn.beforestatic.example.com → cdn.static.example.com
Strip trailing slash/$(empty)replaceRemoves trailing slash
Redirect subdomain^https://old\.https://new.replaceold.site.com → new.site.com

JSON Format

Import/export uses a JSON array. Each rule: id, name, pattern, modification, mode ("before"/"replace"/"after"), enabled.

Firefox Android Install

  • In Firefox for Android, go to Settings → About Firefox and tap the Firefox logo 5× to enable debug mode.
  • Then go to Settings → Install extension from file and select the .xpi (zip) file.

Chrome / Edge / Brave Install

  • Go to chrome://extensions (or edge://extensions)
  • Enable Developer mode
  • Click Load unpacked and select the extension folder
